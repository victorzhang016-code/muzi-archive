# 模子の衣柜 — 承载力 / 容量评估

> 评估当前**免费档**架构（Firestore free-tier 命名库 + Vercel Hobby + Phase1 边缘缓存 + Phase2 图片拆分）面向公众时能扛多少。
> 数字基于 2026-06 实测（见文末「数字来源」）。**面向公众 / 扩容 / 定价 / 升级决策前先看本文。**
> 更新日期：2026-06-15

---

## 结论速览（TL;DR）

| 使用形态 | 免费档稳扛 | 先撞哪堵墙 |
|---|---|---|
| **自用为主**（各管各的柜，少量公开）| **~100–150 注册用户 / ~150–300 日活** | 存储(1GiB) + 实时读(5万/天) |
| **公开爆款**（单个柜被带火，海量人看）| **每月 ~5万–10万 次浏览**（单品分享链接可达 ~20万）| Vercel 带宽(100GB/月) |

> 核心机制：**公开访问走 CDN 缓存，Firestore 读取与访客数无关**（实测热缓存 0 读）。所以「多少人看」主要吃**带宽**，不吃 Firestore 读。

---

## 假设（基于实测，换规模时按需调整）
- 每件单品的 base64 图 ≈ **85 KB**；公开整柜元数据（去 base64 后）≈ **92 KB**；单张图经 `/api/img` ≈ **67 KB**。
- 一次「完整公开衣柜浏览」≈ **~2 MB**（app 包 + 元数据 + 滚动看到的若干图）。
- 一次「单品/搭配深链浏览」≈ **~480 KB**。
- 每位用户约 **100 件单品**（Victor 当前 101 件）。
- 每位登录用户开一次 app ≈ 读自己 **~140 条**（柜 ~100 + 搭配 ~40，onSnapshot 初始拉取）。

## 三道天花板（免费档）

| 资源 | 免费额度 | 谁在消耗 |
|---|---|---|
| **Firestore 读** | **5 万次/天**（此命名库硬上限，开账单也提不了）| 登录用户实时读自己的柜 + 公开页缓存刷新 |
| **Firestore 存储** | **1 GiB** | base64 图片（每件 ~85KB）|
| **Vercel 带宽** | **100 GB/月**（≈3.3GB/天）| 每个访客下载的页面 + 图片 |

> 注：Vercel **函数调用次数 / 边缘请求**不是瓶颈——缓存命中(HIT)不触发函数；只有缓存 MISS（回源 / 部署后冷启）才调用函数，量很小。

---

## 场景 1：自用型增长（大家各管各的衣柜，公开少）
**瓶颈 = 存储 + 登录用户实时读**

- **存储**：1 GiB ÷ 85KB ≈ **~12,000 件单品**。换算：
  - **~120 人**（每人 100 件）
  - **~600 人**（每人 20 件）
  - → 通常是**注册量第一堵墙**。
- **实时读**：5 万/天 ÷ 140 读每人每次 ≈ **~350 日活**（每人每天开一次）；重度多开则降到 ~150。
- **结论**：免费档稳扛 **~100–150 注册用户 + ~150–300 日活**，先撞存储。

## 场景 2：公开爆款（某个衣柜被小红书带火，海量人看）
**瓶颈 = Vercel 带宽（Firestore 完全够）**

- **Firestore**：单个爆款作者不管多少人看，每天只花 **~1k–4k 读**（缓存每小时刷一次 + 图片每天刷一次）→ 5 万额度绰绰有余。
- **带宽**：
  - 完整衣柜浏览 ~2MB → 100GB ÷ 2MB ≈ **~5 万次/月**（~1,600/天）。
  - 单品分享链接 ~480KB → 100GB ÷ 480KB ≈ **~20 万次/月**。
  - 重复访客更省（图片走浏览器+CDN 缓存）。
- **结论**：单个爆款衣柜稳扛 **每月 ~5万–10万次浏览**，Firestore 毫无压力。

---

## 想扩 10–100 倍：按性价比顺序解开
1. **图片搬出 Firestore → 对象存储 / CDN（Cloudflare R2 / S3）**
   - 一举解开「存储 1GiB」+「带宽」两道墙；base64 存库是当初为绕 Storage 不可写的妥协（见 `踩坑经验.md` 坑 1/15），**规模化必须拆**。
2. **Firebase 升 Blaze + 迁标准库**
   - 解开 5万读/天 + 1GiB 存储硬上限（按量付费，个人量级很便宜）。注意当前 `ai-studio-...` 命名库是 free-tier 硬上限，**升 Blaze 也提不了，需迁到标准库**（见坑 3/6）。
3. **Vercel 升 Pro（$20/月，1TB 带宽）**
   - 真撞带宽再升。

## 数字来源（实测，2026-06）
- 公开接口 `/api/public/:uid` 整柜元数据：**92 KB**（Phase2 去 base64 后；Phase2 前是 8.58 MB）。
- 单张图 `/api/img/:uid/:id`：**67 KB**，`x-vercel-cache: HIT`。
- 稳态实测：热缓存下打开公开页，metadata + 图片全部 `x-vercel-cache: HIT`，**Firestore 0 读**。
- Firestore 免费层每日读上限由 429 报错确认：`Free daily read units per project (free tier database)` = 50,000。
- 相关架构与坑：见 `踩坑经验.md`（坑 6 额度、15/16/17 base64+缓存+图片拆分）与 `CLAUDE.md`「公开页边缘缓存」节。
